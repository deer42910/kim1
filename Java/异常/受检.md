**捕获**

出现了异常之后，自己内部解决，消化完之后逻辑上就当解决了，程序还可以向下执行，最终返回调用处。

对于调用者来说，感知不到异常的出现和解决的过程。

**上抛**

程序出现异常之后，放弃未执行的代码，立即返回到调用处，并把问题抛出给调用者。

问题没有得到解决。原有的既定流程，代码无法继续执行下去。
